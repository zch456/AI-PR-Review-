# GitHub PR 元信息获取实施计划

> 给后续执行者：本计划必须按 TDD 执行。每个功能点先写失败测试，再写最小实现，再跑测试通过。步骤使用 `- [ ]` 复选框记录状态。

**目标：** 在已有 PR URL 解析闭环基础上，接入 GitHub API 获取公开 PR 的基础元信息，并在后端接口和前端页面中展示。

**架构：** 后端新增一个独立的 GitHub 客户端模块，负责调用 GitHub REST API 并映射为内部数据结构。`/api/analyze-pr` 仍然作为前端入口，先解析 URL，再获取 PR 元信息，最后返回结构化响应。前端只展示元信息，不获取 diff、不做风险识别、不调用 AI。

**技术栈：** Python、FastAPI、Pydantic、httpx、pytest、React、TypeScript、Vite。

---

## 范围

本阶段实现：

- 后端 GitHub PR 元信息客户端。
- `/api/analyze-pr` 返回 PR 标题、作者、状态、分支、增删行和变更文件数量。
- 前端展示 PR 元信息。
- README 补充 GitHub API 说明。

本阶段不实现：

- changed files 列表。
- patch diff 获取。
- 风险规则。
- AI Review。
- Markdown 导出。

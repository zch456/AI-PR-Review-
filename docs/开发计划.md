# 开发计划

## 当前状态

项目已经完成初步可用版本。用户可以输入公开 GitHub PR 链接，系统会获取 PR 元信息、变更文件、patch，执行规则化风险预检查，并可调用 DeepSeek 生成结构化 AI Review 报告。

当前能力包括：

- GitHub PR URL 解析。
- 公开 PR 元信息获取。
- changed files 和 patch 获取。
- 规则化风险预检查。
- DeepSeek AI Review 生成。
- 前端结构化展示。
- Markdown 报告导出。

## 当前架构

```text
浏览器
  -> POST /api/analyze-pr
    -> pr_url_parser
    -> github_client
    -> risk_engine
  <- PR 元信息、变更文件、风险提示

浏览器
  -> POST /api/generate-review
    -> pr_url_parser
    -> github_client
    -> ai_reviewer
  <- AI Review 总结和发现项
```

后端模块分工：

- `main`：FastAPI 路由和错误转换。
- `pr_url_parser`：解析 GitHub PR 链接。
- `github_client`：获取 PR 元信息和文件变更。
- `risk_engine`：执行确定性风险规则。
- `ai_reviewer`：构建 prompt、调用 DeepSeek、解析模型输出。
- `config`：读取环境变量配置。
- `schemas`：定义 API 请求和响应结构。

前端目前集中在 `web/src/App.tsx`，已经能完成完整使用流程。后续如果继续扩展，应拆分组件，降低单文件复杂度。

## 近期计划

### 1. 配置和错误体验优化

目标：

- 增加 `.env.example`，明确 DeepSeek 和 GitHub token 配置。
- 后端启动时或请求失败时给出更清楚的配置状态提示。
- 前端区分 GitHub 错误、AI 配置错误、AI 超时和模型输出格式错误。

验收标准：

- 用户不看源码也能完成配置。
- 缺少 `DEEPSEEK_API_KEY` 时，页面提示明确。
- README 和 `.env.example` 保持一致。

### 2. 上下文构建升级

目标：

- 从“按变更行数排序拼接 patch”升级为更可控的上下文构建。
- 对大 PR 分文件分批分析。
- 对无 patch 的文件、二进制文件、lock 文件做特殊处理。

验收标准：

- 大 PR 不会因为输入过大直接失败。
- 模型回答会说明分析范围和省略内容。
- prompt 构建逻辑有单元测试覆盖。

### 3. Review 报告结构增强

目标：

- 把 AI 输出从 `summary + findings` 扩展为更完整的报告结构。
- 增加 `recommendedTests`、`reviewQuestions`、`riskLevel`、`keyChanges`。
- 区分“确定问题”和“需要人工确认的问题”。

验收标准：

- 后端 schema 和前端展示同步更新。
- 低置信度问题不会被包装成确定缺陷。
- Markdown 导出覆盖完整报告结构。

### 4. 私有仓库和限流支持

目标：

- 支持 `GITHUB_TOKEN`。
- 对 GitHub 401、403、404、rate limit 做更细错误提示。
- README 明确 token 权限边界。

验收标准：

- 公开仓库无需 token 仍可使用。
- 私有仓库配置 token 后可访问。
- token 不会出现在日志、前端响应或提交记录中。

### 5. 前端组件化

目标：

- 拆分 `App.tsx`。
- 独立出 PR 输入、元信息摘要、风险列表、文件列表、AI Review、Markdown 导出等组件。
- 增加清晰的加载状态和空状态。

验收标准：

- 单个组件职责清晰。
- 页面在移动端和桌面端都可读。
- 后续加功能不需要继续堆大文件。

## 中期计划

- 支持将 Review 建议发布回 GitHub PR。
- 增加用户可配置的规则集。
- 引入示例 fixture，方便无网络演示。
- 增加端到端测试。
- 支持 Gitee 或 GitLab。

## 暂不处理

这些能力暂时不进入近期开发：

- 多用户登录。
- 团队仪表盘。
- 数据库存储 Review 历史。
- GitHub App 完整安装流程。
- 仓库级语义索引。

这些方向价值更高，但工程量也更大，建议等当前单用户工具稳定后再推进。
